PASO 1: CREAR .env LOCAL
────────────────────────────────────────────────────────────────────────────

  cd "/home/ixcsoft/Dashboard- Hola suite"
  cp .env.example .env

Abre .env en tu editor (nano, vim, o VS Code):

  nano .env

Busca estas líneas y completa:

  SUPABASE_URL=https://vvrnufzwyxszknnnfvig.supabase.co
  
  SUPABASE_SERVICE_KEY=eyJ...  ← PEGA AQUÍ LA KEY DE SUPABASE
  
  SUPABASE_ANON_KEY=eyJ...     ← PEGA AQUÍ LA OTRA KEY

PASO 3: AGREGAR VARIABLES EN VERCEL
────────────────────────────────────────────────────────────────────────────

Abre tu navegador:
1. Ve a https://vercel.com/dashboard
2. Busca y haz clic en proyecto "hola-suite"
3. Ve a "Settings" (arriba)
4. En la izquierda: "Environment Variables"
5. Agrega CADA una de estas:

   ┌──────────────────────────────────────────────────┐
   │ Variable Name: SUPABASE_URL                      │
   │ Value: https://vvrnufzwyxszknnnfvig.supabase.co  │
   │ [Save]                                           │
   └──────────────────────────────────────────────────┘

   ┌──────────────────────────────────────────────────┐
   │ Variable Name: SUPABASE_SERVICE_KEY              │
   │ Value: eyJ... (la key grande de Supabase)        │
   │ [Save]                                           │
   └──────────────────────────────────────────────────┘

   ┌──────────────────────────────────────────────────┐
   │ Variable Name: SUPABASE_ANON_KEY                 │
   │ Value: eyJ... (la otra key de Supabase)          │
   │ [Save]                                           │
   └──────────────────────────────────────────────────┘

   ┌──────────────────────────────────────────────────┐
   │ Variable Name: JWT_SECRET                        │
   │ Value: algo_super_seguro_1234567890              │
   │ [Save]                                           │
   └──────────────────────────────────────────────────┘

PASO 4: REDEPLOY EN VERCEL
────────────────────────────────────────────────────────────────────────────

En Vercel Dashboard:
1. Click en "Deployments" (arriba)
2. Busca el deployment más reciente
3. Haz clic en los 3 puntos "..."
4. Selecciona "Redeploy"
5. Espera a que termine

O más fácil:

  Simplemente haz push a GitHub nuevamente:
  
    git push origin main
  
  Vercel se redesplegará automáticamente.

🆘 SI NO FUNCIONA:
────────────────────────────────────────────────────────────────────────────

Revisa los logs en Vercel:

  vercel logs --follow

O en Vercel Dashboard:
1. Click en último deployment
2. Busca "Logs"
3. Busca mensajes de error

Problemas comunes:

  ❌ "Cannot find module @supabase/supabase-js"
  ✅ Solución: npm install @supabase/supabase-js

  ❌ "SUPABASE_URL is undefined"
  ✅ Solución: Verifica que agregaste las variables en Vercel

  ❌ "Connection refused"
  ✅ Solución: Verifica que las keys de Supabase son correctas

Lee VERCEL_FIX.md para más detalles.
