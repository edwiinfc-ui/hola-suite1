P: ¿Cómo cambio la contraseña de un usuario?
R: Las contraseñas están hasheadas con bcryptjs. Para cambiar:
   1. hash = bcryptjs.hashSync(newPassword, 10)
   2. Actualizar en users.json
   3. Reiniciar servidor

P: ¿Los datos se sincronizan automáticamente?
R: Actualmente sincronización manual (POST /api/sync/start)
   Los datos se cachean y se actualizan cada 1 hora
   Para automática: Agregar setInterval en server.js

P: ¿Qué pasa si ClickUp no responde?
R: RetryWithBackoff reintentra automáticamente (máx 3 veces)
   Si falla: Fallback a data/clientes.json (local cache)
   El cache se invalida cuando los datos cambian

P: ¿Puedo usar las integraciones opcionales?
R: Sí, son 100% opcionales. El sistema funciona sin ellas
   Solo agregar credenciales a .env para activarlas

P: ¿Cómo reporto un bug?
R: Contacta al equipo técnico con:
   1. Descripción del problema
   2. Pasos para reproducir
